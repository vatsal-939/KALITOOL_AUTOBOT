7. Echo Client/Server & Advanced Utility Service

Syntax:
nping --echo-client <passphrase> [options] <target>
nping --echo-server <passphrase> [options] <target>

Options:
where ever see | means they cannot be combined.

⦁	Echo Client/Server Mode
--echo-client <passphrase>
--echo-server <passphrase>
--echo-port <port>
--no-crypto
--once
--safe-payloads

-> --echo-client: Run Nping in client mode for echo protocol
-> --echo-server: Run Nping in server mode for echo protocol
-> --echo-port: Use custom port for echo client/server (default uses standard ports)
-> --no-crypto: Disable encryption and authentication for echo protocol
-> --once: Stop echo server after one connection (server mode only)
-> --safe-payloads: Erase application data in echoed packets (security feature)

⦁	Output Control (Verbosity & Debugging)
-q | -q[N]
--quiet
--debug

-> -q: Decrease verbosity level by one (can be used multiple times: -q, -qq, -qqq)
-> -q[N]: Decrease verbosity level N times
-> --quiet: Set verbosity and debug level to minimum
-> --debug: Set verbosity and debug to maximum level (opposite of --quiet)

⦁	Informational & Help
-h | --help
-V | --version

⦁	Packet Capture Filtering
--bpf-filter <filter spec>

-> Specify custom BPF (Berkeley Packet Filter) filter for packet capture
-> Allows fine-grained control over which packets are captured/processed

⦁	Target Specification Options (for {target specification})
command-line targets (hostname, IP, CIDR, range) | -iL <inputfile> | -iR <num hosts>
--exclude <hosts>
--excludefile <file>

-> These options apply to all probe modes and echo modes
-> -iL: Input from list of hosts/networks
-> -iR: Choose random targets (number of hosts)
-> --exclude: Exclude specific hosts/networks
-> --excludefile: Exclude list from file
-> Exclude options ignore single host targets

⦁	Additional Notes on Echo Mode
-> Echo client/server mode allows testing network connectivity with encrypted echo protocol
-> Requires matching passphrase on both client and server sides (unless --no-crypto used)
-> Can be combined with probe mode options (e.g., --tcp, --udp, --icmp) for echo over different protocols
-> Echo protocol useful for network testing, firewall rule verification, and connectivity checks
-> See examples in help: --echo-server "public" -e wlan0 -vvv

⦁	Compatibility Notes
-> Echo modes work with probe mode options (--tcp, --udp, --icmp) when specified
-> Most other probe-specific options (e.g., --flags, --icmp-type) can be combined with echo modes
-> Output and verbosity options work with all modes including echo
-> See nping man page for more options, descriptions and usage examples

=> Echo modes can be used standalone or combined with probe mode options
=> Output control options can combine with any service (1-6) and echo modes

⦁ Echo ≠ Ping
-> Echo protocol:
        -> Is custom, not ICMP echo
        -> Requires both sides running nping
-> Firewalls may:
        -> Allow ICMP
        -> Block echo protocol

⦁ --no-crypto
-> Completely disables:
        -> Authentication
        -> Encryption
-> Should be flagged as unsafe in automation UI

⦁ --safe-payloads
-> Strongly recommended when:
        -> Echoing application data
        -> Testing in shared environments


⦁ Cross-Service Interaction Notes (Important)
Echo + Probe Modes
-> Valid combinations: Echo over TCP / UDP / ICMP
-> Invalid assumptions:
        -> Echo does not inherit all probe semantics
        -> Some probe-specific fields are ignored

--bpf-filter
-> Affects capture, not sending
-> Can make it look like:
        -> No responses
        -> Broken network
-> Automation should warn when used
