-PS <ports>
-PA <ports>

-PE

⦁	DNS Resolution Options
-n | -R

--system-dns

⦁	Traceroute Option
--traceroute

-> cannot work with -sL

⦁ -Pn Behavior (Very Important)
-> -Pn does not disable host discovery logic internally
-> It:
-> Skips ping checks
-> Forces all targets into “up” state
-> Side effects:
-> Slower scans
-> More noise
-> More firewall hits

⦁ -sn Clarification
-> -sn:
        -> Disables port scanning
        -> Still allows:
            -> DNS resolution
            -> Host discovery probes
            -> Traceroute (if enabled)

⦁ DNS Resolution Notes
-> --dns-servers:
        -> Overrides system resolver
        -> Affects performance significantly
-> --system-dns:
        -> Disables Nmap’s async DNS engine
        -> Slower but more OS-consistent

⦁ --traceroute
-> Runs after host discovery
-> Generates additional packets
-> Can reveal scanning activity clearly

-> -sU:
        -> Is extremely slow
        -> Produces many open|filtered results
-> When combined:
        -> Nmap silently ignores incompatible scans

        -> Default scan is used

⦁ FTP Bounce (-b)
-> Rarely works today
-> Requires: Misconfigured FTP server
-> Should be marked legacy / experimental

7. Advanced Options & Utility Service

Syntax:
nmap [scan service syntax] [advanced/utility options]

Options:
where ever see | means they cannot be combined.

⦁	Timing & Performance Control
-T<0-5>
--min-hostgroup <size>
--max-hostgroup <size>
--min-parallelism <numprobes>
--max-parallelism <numprobes>
--min-rtt-timeout <time>
--max-rtt-timeout <time>
--initial-rtt-timeout <time>
--max-retries <tries>
--host-timeout <time>
--scan-delay <time>
--max-scan-delay <time>
--min-rate <number>
--max-rate <number>

-> Time formats: append 'ms' (milliseconds), 's' (seconds), 'm' (minutes), 'h' (hours)
-> -T<0-5>: Timing template (0=paranoid, 1=sneaky, 2=polite, 3=normal, 4=aggressive, 5=insane)
-> --min-rate/--max-rate: Control packet sending rate (packets per second)

⦁	Firewall/IDS Evasion & Spoofing
-f | --mtu <val>
-D <decoy1,decoy2[,ME],...>
-S <IP_Address>
-e <iface>
-g <portnum> | --source-port <portnum>
--proxies <url1,[url2],...>
--data <hex string>
--data-string <string>
--data-length <num>
--ip-options <options>
--ttl <val>
--spoof-mac <mac address/prefix/vendor name>
--badsum

-> -f: Fragment packets (optionally with --mtu to specify MTU size)
-> -D: Cloak scan with decoy IPs (use ME to indicate your real IP position)
-> -S: Spoof source IP address (requires root/administrator privileges)
-> -e: Use specified network interface
-> -g/--source-port: Use given source port number
-> --proxies: Relay connections through HTTP/SOCKS4 proxies
-> --data/--data-string/--data-length: Append custom payload to packets
-> --ip-options: Send packets with specified IP options
-> --ttl: Set IP time-to-live field
-> --spoof-mac: Spoof MAC address (can use vendor name like 'Apple')
-> --badsum: Send packets with invalid checksums (IDS testing)

⦁	TCP Scan Flags Customization
--scanflags <flags>

-> Customize TCP scan flags (can combine SYN, ACK, FIN, RST, PSH, URG)
-> Requires root/administrator privileges
-> Used with TCP scan types (-sS, -sT, etc.)

⦁	IPv6 Support
-6

-> Enable IPv6 scanning

⦁	Aggressive Scan (All-in-One)
-A

-> Equivalent to: -O -sV -sC --traceroute
-> Enables OS detection, version detection, script scanning, and traceroute

⦁	Data Directory & Send Method
--datadir <dirname>
--send-eth | --send-ip

-> --datadir: Specify custom Nmap data file location
-> --send-eth: Send using raw Ethernet frames
-> --send-ip: Send using raw IP packets

⦁	Privilege Assumptions
--privileged | --unprivileged

-> --privileged: Assume user has full privileges (default if running as root)
-> --unprivileged: Assume user lacks raw socket privileges

⦁	Informational
-V
-h

-> -V: Print version number
-> -h: Print help summary page

⦁	Additional Notes
-> Many options require root/administrator privileges for raw socket access
-> Firewall evasion options may be detected by advanced IDS/IPS systems
-> Timing options can significantly affect scan speed and stealth
-> See nmap man page (https://nmap.org/book/man.html) for more options and examples

=> Can combine with services 1-6 as appropriate
=> Timing and performance options affect all scan operations
=> Firewall/IDS evasion options typically used with port scanning (Service 2)

⦁ Timing Templates
-> -T4, -T5:
        -> Easily detected
        -> Can cause packet loss
-> -T0, -T1:
        -> Extremely slow
        -> Unsuitable for automation

⦁ Decoys (-D)
-> Only hide source IP
-> Do NOT:
        -> Hide timing
        -> Hide scan patterns
-> IDS can still detect real scanner

⦁ Spoofing Reality
-> -S, --spoof-mac:
        -> Often useless on routed networks
        -> Replies go to spoofed address
-> Should be marked advanced / expert-only

⦁ --send-eth
-> Requires: Same L2 network
-> Breaks across routers
